# 公众号版式骨架协议（wechat_layout_skeleton）

> v0.2.1 新增。排版先有骨架，再做美化。任何文章不得只把 Markdown 转成漂亮段落，必须先整理为可扫读、可记忆、可行动的公众号结构。

## 1. 四条铁律

1. **先骨架，后美化。**
2. **先重点，后装饰。**
3. **先手机端可读，后桌面端好看。**
4. **一篇文章只能有一套视觉语言。**

## 2. 标准文章骨架

每篇文章至少包含以下结构；可按文章类型裁剪，但不得完全省略骨架。

1. **标题区**：标题有判断，不写空话。
2. **开篇结论卡**：先告诉读者这篇文章的核心结论。
3. **问题引入**：说明读者为什么要继续看。
4. **文章导航**：用 2–4 个要点说明本文讲什么。
5. **主体模块 A/B/C**：每个模块一个小标题、一个关键判断、一段解释或案例。
6. **重点金句块**：提炼 1–3 句可转发、可记忆的核心观点。
7. **行动清单 / 方法步骤**：告诉读者看完能做什么。
8. **结尾收束**：一句话总结，可轻量引导关注或互动。

## 3. 版式骨架默认模板

```text
01｜标题区
02｜开篇结论卡
03｜问题引入
04｜文章导航
05｜主体模块 1：判断 + 解释 + 例子
06｜主体模块 2：对比 / 清单 / 步骤
07｜主体模块 3：方法 / 行动建议
08｜重点金句块
09｜行动清单
10｜结尾收束
```

## 4. 内容分层

公众号正文必须分为五层：

| 层级 | 作用 | 视觉权重 |
| --- | --- | --- |
| 主阅读层 | 正文观点、论证、故事、方法 | 最高 |
| 重点层 | 关键判断、金句、结论卡 | 高 |
| 辅助阅读层 | 引用、补充说明、旁注 | 中低 |
| 可信度层 | 来源、参考资料、图片注释、免责声明 | 低 |
| 行动层 | 关注、二维码、往期推荐、引流说明 | 低，不抢正文 |

## 5. 输出门禁

- 没有开篇结论卡：warning。
- 没有文章导航且文章超过 1200 字：warning。
- 主体模块超过 4 个且没有分区标题：warning。
- 重点卡片超过 4 个：warning，重点被稀释。
- 全文只有段落，没有卡片 / 列表 / 小标题：fail。
