# Editorial Layout Beauty Engine（整篇文章视觉编排引擎）v0.3.3

负责“整篇文章怎么排才好看”。

## 视觉节奏规则

一篇好看的公众号文章应该是：

```text
标题区
开头结论卡片
自然段
小标题
自然段
重点句
引用块
自然段
清单卡片
自然段
总结卡片
轻 CTA
```

禁止：

```text
标题
段落
段落
段落
段落
段落
段落
```

也禁止：

```text
标题
卡片
卡片
卡片
卡片
卡片
```

## 组件密度规则

- 每 3-5 个自然段需要一次视觉变化。
- 每篇文章至少有 1 个开头结论卡片。
- 每篇文章至少有 2 个有设计感的小标题。
- 每篇文章最多 4 个重点卡片。
- 引用块不能超过正文段落数量的 30%。
- 清单卡片不能连续出现。
- CTA 只能在文末或明确需要的位置出现。
- 辅助小字不能抢正文。

## 不要模板味

禁止默认生成以下 Skill 元内容或固定模板句：

- 模块 1 / 模块 2
- 本文将从以下几个方面展开
- 你可以这样检查
- 先把短句合并成自然段
- 再把重点句单独拎出来
- 最后处理引用和引流信息
- 如果你想继续看这类内容，可以关注我。后面我会继续拆解公众号排版、Skill 设计和 AI 工作流。

## section_heading_rewriter

小标题不能机械，应根据内容生成自然标题：

- 一句一行导致文章很长 → 为什么一句一行会让文章变长
- 引用和资料来源不同 → 引用观点和资料来源，不是一种东西
- 重点标记太多会很吵 → 重点不是越多越好

## 重点标记美化

- 每段最多 1 个关键词重点。
- 每 500 字最多 2-3 个关键词重点。
- 每 800 字最多 1-2 个重点句。
- 每篇文章最多 4 个重点卡片。
- 清单编号可以加粗，但不要计入正文重点疲劳。

# v0.3.3.1 content_deduplication_engine

- title、digest、intro、emphasis、summary 不得直接复用同一句。
- summary 必须重新压缩，不得复制 intro。
- digest 像导语；intro_conclusion_card 像观点；emphasis_sentence 像重点判断；summary_card 像收束。
- 同一核心句最多出现 2 次，超过则重写。
- actions 只能从原文提取；原文没有行动建议，不生成 checklist。
- 禁止输出通用三连句：先抓住文章的核心判断 / 再看每个部分如何展开 / 最后把方法带回自己的场景。

## Bullet List Visual Hotfix v0.3.3.2

关键判断、问题清单、行动清单中的列表项必须“一行一个真实项目”。禁止输出只有黑点没有文字的空段落；禁止把 `•` 单独放一行、文字另起一行。正确结构是同一个 `<p>` 内包含 `<span>•</span>` 与文本，项目间距用 `margin-bottom` 控制。
