# Visual Director（视觉导演）v0.3.3

> 安全 HTML 是底线，好看才是目标。Visual Director 必须在 `wechat_html_renderer` 之前工作：先规划视觉骨架，再渲染 HTML，不等生成后补救。

## 负责事项

1. 文章视觉气质判断
2. 版式骨架选择
3. 组件密度控制
4. 重点层级规划
5. 小标题样式选择
6. 引用样式选择
7. 卡片样式选择
8. 清单样式选择
9. CTA 样式选择
10. 全文视觉节奏检查

## 视觉气质类型

- `clean_knowledge_magazine`：清爽知识杂志风（默认）
- `warm_editorial`：温和公众号编辑风
- `blue_tech_explainer`：蓝色科技解释风
- `minimal_business`：极简商业分析风
- `book_note_editorial`：读书笔记杂志风
- `course_promo_clean`：课程推广清爽风
- `personal_opinion`：个人观点专栏风
- `tutorial_guide`：教程指南风

## 视觉导演原则

1. 安全 HTML 是底线，好看才是目标。
2. 公众号排版不是把内容塞进盒子。
3. 每篇文章必须有视觉节奏，不允许一路纯段落。
4. 每篇文章也不能一路全是卡片。
5. 小标题、正文、引用、卡片、清单要形成节奏变化。
6. 一篇文章只允许 1 个主强调色。
7. 卡片样式要统一，不允许每个卡片都不同。
8. 重点标记要少而准。
9. CTA 必须轻，不允许破坏正文气质。
10. 最终效果必须像人工编辑排出来的公众号文章。

## 默认视觉方案

- 主强调色：#1746A2
- 背景：白色 + 少量 #F7F8FA / #FFFDF8
- 小标题：问题式 / 左线 / 数字标签 / 短横线杂志式混合，但同文不超过 2 种主标题风格。
- 引用：左线浅底为默认，读书笔记可使用纸张感引用。
- 卡片：圆角 10px，浅底细边，不使用大黑底和强按钮。

# v0.3.3.1 content_deduplication_engine

- title、digest、intro、emphasis、summary 不得直接复用同一句。
- summary 必须重新压缩，不得复制 intro。
- digest 像导语；intro_conclusion_card 像观点；emphasis_sentence 像重点判断；summary_card 像收束。
- 同一核心句最多出现 2 次，超过则重写。
- actions 只能从原文提取；原文没有行动建议，不生成 checklist。
- 禁止输出通用三连句：先抓住文章的核心判断 / 再看每个部分如何展开 / 最后把方法带回自己的场景。


## v0.3.4 手机端扫读规则

- 章节编号不能像页码，必须服务标题。
- 每个章节至少有一个“核心判断”视觉锚点。
- 正文关键词只做温和加粗，不允许整段加粗。
- 每屏强调不超过 2 处，避免高级感被高亮噪音破坏。


## v0.3.4.1 稀疏强调原则

- 视觉锚点不等于每节都上卡片。
- 标题、短线、段首加粗、关键词加粗、引用都可以形成停顿。
- 正文标题不得重复公众号后台标题。
- 如果第一屏已经有“先说结论”，第一节通常不再加重点卡片。


## v0.3.5 智能配图规划

- 图片风格必须由内容决定，不固定套模板。
- 封面图按 2.35:1 横版规划，同时必须兼容中心 1:1 裁切。
- 正文配图按 3:4 规划，短文 1 张，中长文 1-3 张。
- 生图执行默认可选，先输出提示词、插入位置和质检规则。
- 必须加入背景重构法去水印 / 水印处理：仅限自有图或本流程生成图；第三方版权水印不得自动移除。
