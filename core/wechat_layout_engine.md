# 公众号排版引擎（wechat_layout_engine）

> 本引擎负责把用户原始文章，变成符合个人风格的、稳定不变形的公众号安全 HTML。它是审美系统与输出之间的「编排大脑」。

## 1. 输入

- 接收用户原始文章（Markdown / 长文 / PPT 文案 / 讲稿 / 笔记 / AI 输出）。
- 可选接收指定主题与主强调色；未指定时使用个人基础风格（见 `references/wechat-article-style-system.md` 第 3 节）。

## 2. 判断文章类型

| 类型 | 特征 | 首选结构 |
| --- | --- | --- |
| AI 工具介绍 | 介绍某工具能力 / 使用 | 开场痛点 → 是什么 → 怎么用（步骤卡）→ 适合谁 → 总结 + 行动 |
| Skill 发布说明 | 发布 / 更新某 Skill | 一句话定位 → 能力清单 → 使用方法 → 版本变更 → 行动 |
| 方法论文章 | 提出一套方法 / 框架 | 结论前置 → 原理 → 步骤 → 案例 → 总结金句 |
| 教程文章 | 手把手教学 | 目标 → 准备 → 步骤卡（有序）→ 常见问题 → 总结 |
| 案例复盘 | 复盘某事件 / 项目 | 背景 → 做了什么 → 结果 → 对比卡 → 可复用经验 |
| 热点评论 | 对热点表态 | 事件 → 判断（金句）→ 分析 → 理性边界 → 总结 |
| 产品说明 | 介绍产品 / 功能 | 一句话价值 → 核心能力卡 → 适用场景 → 行动 |
| 清单文章 | N 个要点 / 工具 | 开场 → 编号条目（统一卡）→ 总结 |

## 3. 根据类型选择排版结构

- 从 `templates/wechat-article-template.md` 选取对应模块组合。
- 保持同一类型结构稳定，不每篇随意变样（这是「个人风格稳定」的关键）。

## 4. 自动重写文章结构

1. **长段拆短**：每段聚焦一个意思，超过 4–5 行拆开。
2. **小标题加强**：为每个逻辑块加「路标式」小标题。
3. **重点句提取**：每段最多提炼一句金句，用金句卡 / 加粗突出。
4. **列表整理**：并列信息转为列表或卡片，不塞进一段。
5. **总结前置或后置**：结论型文章总结前置，叙事 / 教程型后置。
6. **结尾给行动建议**：结尾给可执行的下一步或关注引导。

## 5. 输出微信公众号兼容 HTML

- 严格遵守 `core/wechat_html_rules.md`，所有样式内联，数值取自 `core/wechat_design_tokens.md`。
- 顶层用一个 `<section>` 包裹。

## 6. 输出前执行兼容性检查

- 在交付前调用 `core/wechat_validator.md`，输出 pass / warning / fail 报告。
- 存在 fail 项时不得交付，先修复再输出。
- 与现有 6 阶段流水线的关系：本引擎是面向「个人风格稳定输出」的轻量排版通道，与 `pipeline.yaml` 的渲染 / 校验阶段兼容，共用同一套 token 与硬规则。


## 7. v0.2.1 版式骨架优先级

排版引擎必须先执行 `core/wechat_layout_skeleton.md`，再执行视觉美化。

执行顺序：

1. 判断文章类型。
2. 抽取核心结论。
3. 生成开篇结论卡。
4. 生成文章导航。
5. 将主体拆为 2–4 个模块。
6. 提炼 1–3 个重点金句块。
7. 生成行动清单或结尾收束。
8. 最后套用视觉 token。

禁止直接把 Markdown 段落转换为 HTML 后交付。没有结构骨架的漂亮排版判定为失败。

## 8. v0.2.1 辅助信息处理

排版引擎必须调用 `core/wechat_auxiliary_info_protocol.md` 对正文以外的信息降级：

- 引用、来源、注释、参考资料、免责声明、作者介绍、二维码说明、引流文案使用 12px–13px。
- 图片注释默认 12px、浅灰、图片下方。
- 非核心引用默认 13px、浅灰、左侧浅灰竖线。
- 文末引流默认小卡片，不使用正文大段字号，不使用强营销按钮。
- 只有核心金句可进入重点卡片；普通引用不得做大卡片。
