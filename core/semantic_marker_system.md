# Semantic Marker System｜语义编辑标记系统

## 核心原则

标记是阅读语法，不是装饰。语义识别只决定原文如何显示，绝不生成新文字、标签、总结或判断。没有明确语义时保留普通正文。

1. 下划线 = 轻强调。
2. 删除线 = 修正旧认知。
3. 原文色块 = 原句原样进入浅色背景和左侧竖线；不增加标题、标签或图标文字。
4. 高亮 = 段内关键词。
5. 竖线引用 = 可截图金句或完整引用。
6. 胶囊标签 = 禁止自动生成。
7. 双栏对比 = 默认关闭，禁止重组原文。
8. 批注旁白 = 只保留原文已有批注。
9. 数据强调 = 数字放大。
10. 分隔线 = 节奏控制。
11. 渐变重点条 = 默认关闭。
12. 编号 = 只保留原文已有编号。

## 语义与颜色

| 语义 | 首选样式 | 首选颜色 |
| --- | --- | --- |
| 知识、概念、方法论 | 蓝色下划线或蓝色高亮 | 主题主色 |
| 改善、行动、解决方案 | 绿色高亮或浅绿块 | #2F855A |
| 提醒、临界点 | 橙色高亮或浅橙块 | #B7791F |
| 风险、错误、禁止 | 红色下划线或浅红块 | #C53030 |
| 洞察、反常识 | 紫色下划线或浅紫块 | #6B46C1 |
| 旧认知、过时方法 | 灰色删除线 | 主题灰色 |
| 补充说明 | 灰色批注 | 主题灰色 |

文章级颜色规划器最多允许主题主色加两种额外语义色。角色未进入活动色集合时，降级为主题主色或灰色，不额外引入颜色。

## 组件触发

### 下划线

用于核心概念、重要判断、关键变量、风险短语。只标记短语，不给整段加线。

### 删除线

只在文本明确出现“你以为、过去常说、旧做法、常见误区、表面答案”等修正语义时使用。不能为了制造反差擅自给普通陈述加删除线。

### 原文重点色块

只允许把原文完整句子在原位置转换为浅色背景与左侧竖线。短文最多 1 个，中长文最多 2 个；不得新增标题、标签、图标文字，不得复制或移动原句。空色块或无法逐字回溯到原文的色块直接阻断 Release。

### 段内高亮

用于行动词、提醒词和数据。每段最多 1 种行内标记类型；同一句禁止同时使用删除线、下划线和高亮。

### 左侧竖线引用

完整保留成对引号。引用不得拆成前后两个组件，也不得把右引号移动到下一段。

### 胶囊标签和图标

标签只做路标：核心判断、常见误区、真正问题、行动建议、风险提醒、一个例子、编辑注、适合人群、不适合人群。图标规则：× 错误，✓ 推荐，! 提醒，? 问题，# 数据，▍ 重点。

### 双栏对比

识别“你以为/真正的问题”“表面上/实际上”“错误做法/正确做法”。使用安全 table，不使用 flex 或 grid。左栏灰色或浅红，右栏主题蓝或浅绿。左右内容必须来自原文。

### 渐变重点条

用于开篇主判断、章节核心结论或结尾收束。整篇最多 1 个。避免把普通句子做成渐变块。

### 批注式旁白

识别“编辑注、编者按、补充说明、边界说明”。最多 1 个，字号 13px。渲染时去掉重复前缀，但保真报告仍追踪原句。

### 数据强调

识别百分比、年份、时间、金额、人数、次数和倍数。一个段落可标记同一类型的最多两个数字，不与其他行内标记叠加。

### 分隔线

- 短线居中：普通停顿。
- 点状线：风险或轻转折。
- 渐变线：知识或洞察升级。
- 带标签线：进入行动或提醒模块。

### 重点句编号

编号和标题或内容必须位于同一个 `<p>` 或同一个卡片单元。禁止单独输出 `01` 再另起一段显示标题。

## 密度门禁

- 每段最多 1 种行内标记。
- 整篇最多 3 种可见强调色。
- 渐变重点条最多 1 个。
- 双栏对比短文最多 1 个，长文最多 2 个。
- 批注最多 1 个。
- 胶囊标签不能每段出现。
- 标记不能改变、删减或补造原意。

## v0.4.3 大序号与矩形文字框

- 一级大标题使用大号 `01 / 02 / 03`，序号、标题和短横线位于同一 section，禁止标题左竖线。
- 矩形文字框使用完整描边和顶部色条，内部只放原文完整句子，不增加标签或图标文字。
- 短文最多 1 个矩形框，中长文最多 2 个；与填充色块交替使用，避免连续堆叠。
- 复合句语义以转折后的核心落点为准；“可以”不得覆盖“但是 / 而 / 真正 / 本质 / 关键”等判断语义。
