# Article Type Layout Recipes｜文章类型排版骨架

version: 0.3.3.1

## tutorial_guide
适合：操作教程、步骤说明、方法清单、工具使用。
骨架：标题区 → 适用人群 / 使用场景 → 步骤导航 → 数字小标题 → 自然段 → 步骤卡片 → 注意事项卡片 → 总结卡片 → 轻 CTA。
特点：数字标签小标题、步骤卡片、少引用、多清单、重点清楚。

## opinion_editorial
适合：观点文、评论、个人判断、趋势分析。
骨架：标题区 → 反常识开头 → 核心判断 → 3 个理由 → 反方解释 → 重点句 → 引用块 → 结论 → 轻 CTA。
特点：左线小标题、重点句、引用块、不要太多卡片、保持专栏感。

## knowledge_explainer
适合：概念解释、方法论、AI 工具说明、Skill 介绍、知识科普。
骨架：标题区 → 常见误解 → 正确认知 → 原理解释 → 例子 → 应用建议 → 总结卡片。
特点：问题式小标题、开头结论卡、引用 / 解释卡、少量清单。

## case_story
适合：故事、案例、复盘、人物经历、转变过程。
骨架：标题区 → 场景开场 → 冲突 → 转折 → 认知变化 → 案例细节 → 原话引用 → 复盘卡片 → 总结。
特点：自然段更多、卡片更少、引用用于原话、重点句用于转折、不要像教程。

## course_promo_clean
适合：课程介绍、训练营、活动说明、产品说明。
骨架：标题区 → 痛点 → 结果承诺 → 课程内容 → 适合人群 → 课程亮点 → 时间地点小字 → 报名 CTA 轻卡片 → 注意事项。
特点：不能像硬广；CTA 可以出现，但必须轻；课程信息可以卡片化；时间地点用小字辅助。

## skill_launch
适合：AI Skill 发布、工具发布、项目介绍、GitHub 项目说明。
骨架：标题区 → 问题 → 我的解决方案 → 功能演示 → 适合谁 → 怎么使用 → 项目地址 / 下载方式 → 下一步计划。
特点：清爽科技风、蓝色主强调、功能清单卡、使用步骤卡、GitHub / 链接区用轻 CTA。
