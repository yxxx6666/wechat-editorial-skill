# Component Beauty Library（视觉组件美化库）v0.3.3

组件从“能用”升级为“好看”。禁止默认输出“模块 1 / 模块 2 / 第一部分 / Section 1”。除非用户原文明确写了这些词，否则不得生成。

## 小标题组件

### 样式 A：数字标签式
适合教程、步骤、知识拆解。编号作为节奏标签，不等于机械“模块”。

### 样式 B：左线标题式
适合观点文、分析文。左侧 4px 主色线，标题 19px。

### 样式 C：胶囊标签式
适合轻知识、课程、清单。胶囊只作为短标签，标题仍保持清晰。

### 样式 D：短横线杂志式
适合深度文、读书笔记。用 32px 短横线制造杂志感。

### 样式 E：问题式小标题
适合解释类文章。上方小字“问题 / 关键 / 方法”，下方 20px 正标题。

## 引用组件

### 引用样式 A：左线浅底
默认引用：浅灰底、左侧主色线、15px。

### 引用样式 B：居中金句
适合短金句：居中、17px、主色、加粗，但全文最多 1 次。

### 引用样式 C：纸张感引用
适合读书笔记：#FFFDF8 纸张底、浅棕边框、圆角 10px。

### 引用样式 D：小注释引用
适合弱引用：上下细线，14px，不抢正文。

## 卡片组件

### 结论卡片
用于开头先说结论。浅灰底 + 细边 + 圆角 10px + 14px 标签 + 16px 正文。

### 重点卡片
浅蓝底，不做强按钮，不用高饱和大色块。

### 清单卡片
使用 section + p + span，不用 ul/ol/li。编号加粗但不计入正文重点疲劳。

### 总结卡片
纸张感 #FFFDF8 + 浅棕边框 + 轻标签。

### 轻 CTA 卡片
只在用户内容有 CTA 信号时出现。细分割线 + 12px 小字，不做大按钮。

## Bullet List Visual Hotfix v0.3.3.2

关键判断、问题清单、行动清单中的列表项必须“一行一个真实项目”。禁止输出只有黑点没有文字的空段落；禁止把 `•` 单独放一行、文字另起一行。正确结构是同一个 `<p>` 内包含 `<span>•</span>` 与文本，项目间距用 `margin-bottom` 控制。


## v0.3.4 章节编号与重点句组件

### 章节编号标题
- 编号 `01 / 02 / 03` 必须和标题绑定在同一个 `<section>` 中。
- 禁止单独输出只有编号的 `<p>`，避免在手机端看起来像页码。
- 编号可以是胶囊、上方小标签或标题前 inline badge，但视觉上必须属于标题。

### 核心判断卡片
- 每个章节默认 1 个核心判断卡片。
- 使用浅蓝底、左侧主色线、圆角 8px、标签“核心判断”。
- 只放一句强判断，不放大段正文。
- 重点句卡片比普通加粗优先级更高，用来制造每 1-2 屏的视觉停顿。


## v0.3.4.1 自然编辑节奏修复

- “核心判断”卡片不再每节强制出现。
- 整篇默认最多 1 个章节重点卡片；开头“先说结论”已经承担总判断功能。
- 普通章节重点优先用段内加粗，而不是卡片。
- 标签必须随语义变化：反直觉点 / 关键提醒 / 研究发现 / 可以先做 / 这一节的重点。
- 连续两个章节不得出现同款蓝色重点卡片。
