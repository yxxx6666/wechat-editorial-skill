# references/image-rules.md

## 图片资产规则

1. 图片负责氛围、场景和记忆点。
2. 正文观点必须由 HTML 文本表达（不靠图片承载核心信息）。
3. 不把核心中文标题写进生成图片。
4. 封面图推荐 21:9。
5. 文内图推荐 16:9。
6. 行动清单图可以用 3:4。
7. 分享卡可以用 1:1。
8. 图片必须有：用途、比例、插入位置、提示词。
9. WorkBuddy 发布到草稿箱前，图片需要先上传到微信素材库，正文中使用微信素材 URL。
10. 如果图片 URL 不可用，必须使用文字占位提醒，不让草稿箱报错。

## 比例速查

| 用途 | 推荐比例 |
| --- | --- |
| 封面图 cover | 21:9 |
| 文内图 section | 16:9 |
| 行动清单图 action_card | 3:4 |
| 分享卡 | 1:1 |

## 占位符约定

图片 URL 未就绪时，在 `<img src>` 位置使用占位符 `WECHAT_MEDIA_URL_image_01`，并在图下加一行 caption 提醒“待上传微信素材后替换”。绝不使用 base64。


## v0.3.5 公众号封面与正文配图

- 封面图：2.35:1 横版，兼容中心 1:1 裁切。
- 正文配图：3:4，按文章结构自动插入。
- 默认只输出图片提示词和任务清单；自动生图为可选执行层。
- 必须加入背景重构法去水印 / 水印处理：只允许自有图 / 本流程生成图做背景重绘、边缘补全、局部重构、模型水印和伪水印清理；第三方版权水印不得自动移除。
