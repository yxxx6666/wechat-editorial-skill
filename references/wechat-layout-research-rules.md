# WeChat Layout Research Rules｜公众号排版研究落地规则

version: 0.3.3.1
name: WeChat Layout Research Polish
status: 公众号排版研究落地修复版

## 0. 总原则

排版不是装饰，而是降低阅读阻力。

公众号文章不是网页排版，也不是 PPT。它首先是手机上的连续阅读流：单列、短段落、强节奏、多停顿、少装饰。每 3-5 屏必须有一个视觉变化点，但视觉变化必须服务内容，不得硬塞组件。

## 1. 公众号阅读场景

- 手机单列阅读。
- 碎片化阅读。
- 用户快速扫读。
- 中途退出成本很低。
- 阅读依赖节奏和停顿。

默认规则：
1. 公众号文章不是网页排版，也不是 PPT。
2. 它首先是手机上的连续阅读流。
3. 默认单列。
4. 默认短段落。
5. 默认强节奏。
6. 默认多停顿。
7. 默认少装饰。
8. 每 3-5 屏必须有一个视觉变化点。

## 2. 开头区规则

默认骨架：

标题 → 一句话结论 / 反常识钩子 → 问题引入 → 本文你会看到什么 → 进入正文。

对应组件：
- title_block
- intro_conclusion_card
- problem_intro
- article_nav

规则：
1. 800 字以上文章必须有 intro_conclusion_card。
2. 1200 字以上文章建议有 article_nav。
3. 开头结论卡片不能复用标题原句。
4. 开头导语不能写成空泛套话。
5. 不要默认写“本文将从以下几个方面展开”，除非文章确实是教程 / 说明文。

## 3. 正文节奏规则

1. 正文段落建议 60-160 个中文字符。
2. 单个自然段手机端不超过 4-6 行。
3. 连续普通正文段落不超过 4 段。
4. 每 3-5 个自然段需要一次视觉变化。
5. 视觉变化可以是小标题、引用、重点句、卡片、图片、清单。
6. 超过 1200 字必须有明确路标。
7. 超过 1800 字必须有阶段小结。
8. 每一屏最好只有一个阅读焦点。

禁止：连续 6 个以上短段落；连续 5 个以上普通正文段落；全文只有正文段落；全文全是卡片；每句话都单独成段。

## 5. 组件频率

800 字以内：1 个开篇结论卡；2-3 个小标题；0-1 个引用块；0-1 个清单卡；1 个总结卡；CTA 可选。

800-1500 字：1 个开篇结论卡；3-5 个小标题；1-2 个引用块；1-2 个清单 / 重点卡；1 个总结卡；CTA 可选。

1500 字以上：1 个开篇结论卡；1 个文章导航；4-7 个小标题；2-3 个引用 / 重点卡；1-2 个阶段小结；1 个最终总结卡；CTA 可选。

## 6. 文章类型骨架

- tutorial_guide：标题区、适用人群 / 使用场景、步骤导航、数字小标题、自然段、步骤卡片、注意事项卡片、总结卡片、轻 CTA。
- opinion_editorial：标题区、反常识开头、核心判断、3 个理由、反方解释、重点句、引用块、结论、轻 CTA。
- knowledge_explainer：标题区、常见误解、正确认知、原理解释、例子、应用建议、总结卡片。
- case_story：标题区、场景开场、冲突、转折、认知变化、案例细节、原话引用、复盘卡片、总结。
- course_promo_clean：标题区、痛点、结果承诺、课程内容、适合人群、课程亮点、时间地点小字、报名 CTA 轻卡片、注意事项。
- skill_launch：标题区、问题、我的解决方案、功能演示、适合谁、怎么使用、项目地址 / 下载方式、下一步计划。

## 7. Validator 检查项

release 必须满足：P0=0；critical P1=0；schema passed；json valid；text rhythm passed；visual_score >= 88；无 module label；无 Skill 元内容；无 default CTA without signal；无 markdown image leak；无 markdown table pipe leak；无 caption mixed into body；component_tree 不包含 visual_plan；visual_plan 位于顶层 JSON。

## Bullet List Visual Hotfix v0.3.3.2

关键判断、问题清单、行动清单中的列表项必须“一行一个真实项目”。禁止输出只有黑点没有文字的空段落；禁止把 `•` 单独放一行、文字另起一行。正确结构是同一个 `<p>` 内包含 `<span>•</span>` 与文本，项目间距用 `margin-bottom` 控制。


## v0.3.4 排版研究落地规则

- 编号标题：`02` 不能独立漂浮，必须和标题绑定。
- 重点句：每个章节提取 1 句核心判断，优先判断句和反常识句。
- 强调：核心判断用卡片，正文只做少量关键词加粗。
- 节奏：每 1-2 屏至少出现一个视觉锚点。
