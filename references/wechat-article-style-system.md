# 公众号排版风格系统（wechat-article-style-system）

> 本文件是 `wechat-editorial-skill` 的审美主脑。所有排版决策都应回到这里对齐，确保「属于我自己的公众号排版风格」稳定不漂移。

## 1. 审美参考来源矩阵

| 参考来源 | 学习用途 | 提炼到公众号排版的要点 |
| --- | --- | --- |
| **Siteinspire** | 干净、高级、极简、留白充足的网站版式 | 高级感来自**留白与节奏**而非装饰；段落之间留稳定呼吸感；不要拥挤、廉价、花哨；信息密度高但视觉轻 |
| **Lapa Ninja** | Landing Page 首屏结构、卖点区块、CTA、信息模块化 | 把 Landing Page 模块化思路迁移成文章里的**标题区 / 重点区 / 案例区 / 总结区 / 行动区**；每个模块只表达一个清晰意图 |

## 2. 公众号文章整体视觉原则

1. **中文阅读优先**：一切排版服务于中文手机端阅读，不为英文网页审美牺牲可读性。
2. **段落要短**：每段聚焦一个意思，通常 2–4 行；超过就拆。
3. **一屏只传达一个小重点**：手机一屏内不要塞多个并列重点。
4. **标题要有判断**：标题给结论、给立场，不写空泛口号（避免「关于 XX 的一些思考」）。
5. **小标题要像路标**：让读者扫一眼小标题就知道这一段讲什么、走到哪了。
6. **重点句要突出**：用加粗或轻底色卡片承载金句，每段最多一个重点。
7. **引用块要克制**：blockquote 只用于真正的引用或强观点，不当装饰。
8. **分割线要轻**：用浅色细线（`border_light`），不用粗黑线或花式分隔。
9. **列表不要太密**：列表项之间留间距（`list_margin`），不要挤成一坨。
10. **卡片块不要太多**：一篇文章卡片控制在少量，过多则失去「重点」意义。
11. **留白要稳定**：段落、小节、卡片的间距统一来自 token，不每篇随意调。
12. **色彩不超过 2–3 个主视觉色**：正文色 + 标题色 + 一个强调色。
13. **不使用复杂布局**：不用 flex / grid / 定位 / 多列。
14. **不使用会被微信公众号过滤的样式**：见 `core/wechat_html_rules.md`。

## 3. 我的公众号基础风格方向

- **风格关键词**：干净、克制、高级、清晰；适合知识型内容，适合 AI / Skill / 工具 / 方法论文章。
- **背景**：白色或接近白色（`bg_page` 纯白，卡片用极浅灰 `bg_card`）。
- **正文**：深灰色（`text_primary` #2A2E34），不用纯黑，避免压迫感。
- **强调色**：只使用**一种**主强调色（`accent_primary` #1746A2），其浅色版 `accent_soft` 仅作底色。
- **标题**：清楚、有力量，字重加粗、字号分级明显。
- **正文**：短句、短段落，节奏明显，多用可转发金句。
- **模块**：轻卡片、浅色背景、细边框、克制圆角（`card_radius` 8px）。
- **整体感觉**：像一篇清爽的**产品说明 + 知识文章**，不像营销号花哨模板。

## 5. v0.2.1 版式骨架原则

v0.2.0 的审美系统必须服从 v0.2.1 的版式骨架。不要为了“高级感”牺牲内容结构。

- 先输出开篇结论卡，再展开正文。
- 长文必须有文章导航。
- 主体控制在 2–4 个清晰模块。
- 重点句用少量金句卡，不满篇卡片。
- 结尾给行动清单或一句明确收束。

## 6. v0.2.1 辅助信息小字原则

正文以外的信息全部降级：引用、注释、来源、参考资料、作者介绍、引流、二维码说明默认小字浅灰，不抢正文。

视觉目标：正文有力量，辅助信息有秩序，引流不打扰。
